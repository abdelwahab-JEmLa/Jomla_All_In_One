## 1. Résumé exécutif (pour IA vierge)
Ce contexte décrit l'implémentation de la fonctionnalité permettant d'ajouter et d'éditer des présentations vidéo ou images pour les couleurs des produits dans l'application Client JetPack. 
Le but était de :
- Permettre à l'administrateur d'ajouter une nouvelle couleur directement avec le bouton "+🎨 Couleur" dans l'en-tête [A_Compact_Header_App4.kt](file:///D:/AndroidStudioProjects/ClientJetPack/app/src/main/java/Application4/App/Fragment/View/Components/A_Header/View/A_Compact_Header_App4.kt).
- Afficher à côté du bouton de liaison `⛓` deux options d'édition : 🖼 pour choisir une image et 🎥 pour choisir une vidéo depuis la galerie.
- Gérer le nettoyage automatique des fichiers précédents (écrasement/suppression) lors du changement de média pour éviter de surcharger le stockage.
- Utiliser un lecteur vidéo ExoPlayer en boucle lorsque la couleur contenant une vidéo de présentation est affichée en mode étendu dans [Image_Displaye.kt](file:///D:/AndroidStudioProjects/ClientJetPack/app/src/main/java/Application4/App/Fragment/View/ViewS/Views/Image_Displaye.kt).

## 2. Plan d'implémentation détaillé (Step-by-step)
1. **Modèle de données** : Ajouter le paramètre `il_a_une_video_presentaion: Boolean` à la classe `M3CouleurProduitInfos` et l'inclure dans la sérialisation `to_Map()`.
2. **Version Base de Données** : Augmenter la version de Room Database dans `AppDatabase.kt` vers la version `6` pour migrer le schéma de `M3CouleurProduitInfos`.
3. **Bouton d'ajout admin** : Déclarer `onAddNewColor` dans `A_Compact_Header_App4.kt` et appeler ce callback lors du clic sur le bouton "+🎨 Couleur".
4. **Implémentation d'ajout** : Dans `A_Item_Produit_App4.kt`, implémenter `onAddNewColor` pour insérer une nouvelle entité `M3Couleur` avec le type `Nom` et rafraîchir le produit.
5. **Modification du sélecteur réutilisable** : Mettre à jour `FastInit_Outlined_Int_Edite_Modulable_Proto4.kt` pour ajouter les paramètres `onPickImage` et `onPickVideo` et afficher les boutons à côté du bouton lien.
6. **Mise en place des sélecteurs de galerie** : Dans `Lenceur_Vent_Handler_App4.kt`, instancier les lanceurs d'activité pour l'image et la vidéo. Lors du retour d'Uri, supprimer le fichier du média précédent (s'il existe), écrire le nouveau média localement, le téléverser en arrière-plan sur Firebase Storage, mettre à jour `il_a_une_video_presentaion` et synchroniser.
7. **Lecteur vidéo dans Image_Displaye** : Dans `Image_Displaye.kt`, si `il_a_une_video_presentaion` est vrai, instancier `ExoPlayer` en boucle (`AndroidView` wrapping `PlayerView`) en mode étendu. En mode réduit, afficher une miniature Glide classique avec un overlay représentant une icône Play.

- **Version Room** : Ne pas oublier de documenter la migration de schéma dans `AppDatabase.kt` et d'incrémenter le numéro de version sous peine de crash au démarrage.
